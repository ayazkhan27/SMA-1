# SMA-1 figure program — master plan

**Status: design specs only. Do NOT generate yet — a separate command will.**

This folder holds the *ideas* for every paper figure, at the rigor bar set by
MAMMAL (npj Drug Discovery, doi:10.1038/s44386-026-00047-4, in
`paper/reference_papers/MAMMAL.pdf`). The lesson from MAMMAL's Figure 1: a
flagship figure is a **multi-panel composite** (A–E) where each panel uses a
*different visual grammar* — icon stacks over an axis, an architecture schematic
with a typed-token legend, checkmark coverage matrices, and a domain-sectored
radar — unified by one color language. Our figures must do the same: shapes,
flows, directional arrows, real glyphs, color-coded semantics. **No figure may
be a box-and-words flowchart.** The two placeholder figures currently in
`paper/figures/` (`fig_pipeline_overview`, `fig_structure_mapping`) are
explicitly the "lackluster" baseline we are replacing.

## Shared design language (applies to every figure)

**Color semantics (fixed across the whole paper — a reader learns it once):**

| Role | Meaning | Palette anchor |
|---|---|---|
| ENTITY / constant | objects, hosts, blocks | neutral slate `#5a6472` |
| ATTRIBUTE (arity-1) | properties of one object | CMasher `cmr.ocean` light |
| FIRST-ORDER relation | timeout, retry, io… | CMasher `cmr.ocean` mid |
| HIGHER-ORDER relation | cause, enables, before | CMasher `cmr.ocean` dark (the systematicity carrier — always the boldest hue) |
| CANDIDATE INFERENCE | projected/hypothetical | warm red `#b3403a`, **dashed** outline always |
| SMA (our method) | — | `cmr.ocean` deep teal, solid, heaviest line |
| Lexical baselines (BM25/dense/hybrid) | — | grayscale ramp |
| KG baselines (KG-PPR/HippoRAG) | — | muted amber |
| Lattice / ascension | climbing the ontology | dotted violet `#7b5ea7` |

**Conventions:** solid arrow = deterministic dataflow; dashed arrow =
hypothetical/projected; dotted = ascension through the lattice; double-headed
dotted = a correspondence (match hypothesis). Every results panel shows 95% CIs
(paired bootstrap) and is generated ONLY from `reports/confirmatory/*.csv`.
Pre-confirmation numbers are hatched. Every figure carries the version stamp
(date · commit · semantics) already standard in `make_paper_assets.py`.

**Toolchain per figure is named in each spec**, drawn from
`paper/GRAPHICS_STACK.md`. Two production routes:
- **Matplotlib route** (SciencePlots + CMasher + adjustText + brokenaxes) for
  data-driven panels — generated by `make_paper_assets.py`.
- **TikZ/PGFPlots route** for the schematic/mechanism panels that need precise
  vector geometry (DAGs, correspondence bundles, nested-relation trickle-down).
  These compile standalone to PDF and are `\input` into the manuscript; NetworkX
  + `nx.nx_agraph`/`netgraph` can pre-compute layouts that we then render in
  TikZ for camera-ready crispness.
