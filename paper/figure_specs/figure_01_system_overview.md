# Figure 1 — SMA-1 system overview (flagship, 5-panel composite)

**Role:** the one figure a reviewer remembers. Direct structural analog of
MAMMAL Figure 1 (panels A–E, each a different visual grammar, one color
language). Full page width, ~2/3 page tall. Panel letters in bold sans, top-left
of each panel, exactly as MAMMAL.

**Toolchain:** Panels A–C in **TikZ/PGFPlots** (vector schematics, compiled
standalone, `\input` into manuscript). Panels D–E in **Matplotlib**
(SciencePlots + CMasher), generated by `make_paper_assets.py`. Final assembly
in LaTeX via `subfigure`/`tikzpicture` overlay so panel registration is exact
(do NOT bitmap-stitch).

---

## Panel A — "Representation": raw artifact → typed predicate DAG

The conceptual payload: SMA stores *structure*, not text. Show one real BGL/HDFS
log session transforming into a predicate DAG.

- **Left third:** a faux-terminal card with 4–5 real log lines (monospace,
  syntax-tinted timestamps/hosts/templates). A downward solid arrow labeled
  `Tier-0 encoder (deterministic)`.
- **Right two-thirds:** the resulting DAG, laid out by **NetworkX
  `graphviz_layout(prog="dot")`** then hand-rendered in TikZ. Nodes color-coded
  by the shared role palette: slate entities at the bottom (order 0), mid-teal
  first-order relations (`timeout`, `retry`), dark-teal higher-order `cause`
  spanning them (order 2). Arc labels carry argument position `i`. A small
  "order" axis on the far left (0,1,2) aligns node rows to their order — echoes
  MAMMAL panel A's data-size axis as an organizing spine.
- **Callout chip:** `case_id = BLAKE3(canonical s-expr)` with the actual hash
  prefix, plus the s-expr `(cause (timeout R63-M0 treeLink) (retry R63-M0
  treeLink))` in a tinted code box. Communicates content-addressing + audit.
- **Caption micro-point:** "Sub-expressions are hash-consed → a case is a DAG,
  not a bag; this is what makes systematicity a graph property."

## Panel B — "Architecture": the write and read paths

Two horizontal swimlanes sharing the central CASE STORE, color-banded by layer
(matching §2 of the architecture blueprint). This is MAMMAL panel B's role (the
model schematic with the typed-token legend).

- **Write lane (top):** raw artifacts (small domain glyphs: log/code/trace/JSON)
  → ENCODER band → CASE STORE (LMDB cylinder + WAL ribbon) → INDEX (inverted
  postings + HNSW).
- **Read lane (bottom):** query → MAC (content vector → bound-ordered shortlist,
  draw the histogram-intersection as overlapping bars) → FAC (SME match, a tiny
  correspondence glyph) → top-k + **receipts** → LLM gate stamped **"cite or
  abstain"**.
- **Typed-token legend** (bottom-right, like MAMMAL's dot legend): the five node
  roles + candidate-inference red + the three arrow types. One legend serves the
  whole figure.
- **The one-LLM boundary** drawn as a dashed enclosure around only the final
  verbalize step, annotated "the only LLM; never writes facts."

## Panel C — "Matcher": inside the SME core

A zoom — the panel that proves there is real machinery, not a wrapper. Left-to-
right micro-pipeline, each stage a small distinct diagram (not a labeled box):

1. **MH seeding** — base DAG and target DAG side by side, faint double-headed
   dotted correspondence threads between same-functor nodes.
2. **Kernels** — three shaded blobs grouping connected correspondence sets.
3. **Conflict graph → MWIS** — kernels as weighted vertices, red edges =
   mutual inconsistency; the selected independent set highlighted (CP-SAT).
4. **SES trickle-down** — the winning map with γ-weighted evidence flowing from
   the `cause` root downward; render score accumulation as growing bar heights
   on each node (deep nodes visibly score higher → systematicity as arithmetic).
5. **Candidate inference** — one base statement projected into the target as a
   red dashed node tagged `:hypothetical`.

Tool: TikZ with a NetworkX-precomputed layout; the trickle-down bars via
PGFPlots inline.

## Panel D — "Coverage": task × method matrix

Checkmark grid exactly like MAMMAL panels C/D. Rows = tasks (SSB, T1 transfer
[×3 legs], T2 family, T2 triage, T3 BugsInPy, T4 ARN, T5 drift). Columns =
methods (SMA, BM25, Dense, Hybrid-RRF, Hybrid+Rerank, KG-PPR, HippoRAG, Long-
ctx). Filled teal check = evaluated; the SMA column shaded as the protagonist.
A second thin column group flags "deterministic / $0 LLM extraction" with a
coin-strike icon (sets up the cost story, H6). Tool: pandas Styler → matplotlib
table, or great-tables.

## Panel E — "Results": domain-sectored radar

The MAMMAL-1E analog. A radar/spider chart whose sectors are the evaluation
domains (Transfer-Spirit, Transfer-Tbird, Transfer-OpenStack, HDFS-family,
BGL-triage, BugsInPy, SSB-r1, Drift-slope). Overlaid polygons: SMA (deep teal,
solid, heavy), best lexical baseline (gray), best KG baseline (amber). Sectors
faintly background-tinted by domain group (logs / code / synthetic) like
MAMMAL's GE/SM/Protein wedges. Numbers from `reports/confirmatory/`. Tool:
Matplotlib polar; adjustText for sector labels.
