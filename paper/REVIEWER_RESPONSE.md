## A. Positioning, novelty, prior art, and language

**1. Is SMA formally defined; is "functor" a real object?** **SURFACED.** The functor *is* a real typed, serializable object (`sma/ir/schema.py`: `Statement(functor, args, ascension)`), and the encoding is literal (`sma/ontology/mount.py` builds `f_T(x)` for each present term and the higher-order `r(f_s(x), f_o(x))` for each typed relation with both endpoints present). The paper now gives the one-line grammar in Methods ("each present term T on subject x is the unary statement f_T(x); each typed relation (s,r,o) is the higher-order statement r(f_s(x),f_o(x)); a case is the set of such statements") and adds the clarifying clause that "functor" denotes a predicate/term-constructor symbol, **not** a category-theory functor, to pre-empt that reading.

## C. Mechanism specification and hyperparameters

**8. Are index / query-encode / MAC / FAC / scoring specified enough?** **SURFACED.** The mechanism was fully in code but under-specified in Methods. We added the formal equations: the trickle-down score S(m) = σ₀(m)·a(m) + γ·Σ_{p∈parents} S(p) with σ₀ = −log₂p; the MAC admissible bound U(q,c) = s̄·Σ_f cost(f)·min(q_f, c_f) with s̄ = 4.0 in production; the ascension factor a = ρ^dist for dist ≤ δ; and a line on the content-vector schema (functor counts + WL-1 features + δ-bounded ancestor closure).

**9. Hyperparameter sensitivity (ρ, δ).** **SURFACED + SCOPED.** The reviewer's described sweep did not match what we ran, and we now say so honestly. The committed calibration grid (`calibration_grid.csv`) sweeps scorer × normalization × γ∈{0.125,0.25,0.5} × ρ∈{0.90,0.95}; it is now rendered as an Extended Data table. We state plainly that **δ was set per-adapter, not swept**, and that ρ was swept only over {0.90, 0.95}, so we make no claim of an exhaustive hyperparameter sweep.

**10. Universal loader: OWL semantics, reasoner, typed relations, multi-parent, cycles.** **FIXED + SURFACED.** The reviewer is right that "ingests any open ontology" overclaimed. There is **no OWL reasoner**: subsumption is realized at match time by bounded δ-hop ascension over asserted edges. The claim now reads "ingests any open OBO/OWL ontology (is-a hierarchy plus someValuesFrom/allValuesFrom and OBO relationships; no OWL reasoning)," and Methods states that multi-parent DAGs are supported, cyclic is-a is truncated at δ, and OWL axioms beyond subClassOf and existential/universal restrictions (unionOf, intersectionOf, cardinality, property chains, equivalentClass) are not consumed.

**35.
